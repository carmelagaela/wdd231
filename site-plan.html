PharmaCare

A guide to safe and smart medication use — site plan for W05

Site Name

PharmaCare — chosen because it clearly combines "Pharma" (pharmacy/pharmacology) with "Care" (safety, patient-centered health). The name is short, brandable, and easily searchable.

Optional domain suggestion: pharmacare.ph or pharmacare-guide.org (domain availability not required for the course).

Site Purpose

The site aims to educate the public about safe medication practices and the role of pharmacists in healthcare. It is a compact resource hub for:

  • Clear medication summaries (uses, dosage, side effects, warnings) for common OTC and prescription drugs.
  • An interactive "Ask a Pharmacist" / medication log feature so users can track what they take and submit questions.
  • Practical tips for storage, disposal, and identifying interactions.

Planned Pages (site map)

  • Home (index.html) — mission, featured tips, quick search to medication index.
  • Medication Info Center — dynamic cards of 15+ medications loaded from medications.json (or API).
  • Ask a Pharmacist / Medication Log — form to submit questions and log medication entries (stored in localStorage).
  • About & Resources — accessibility statement, sources, disclaimers, and pharmacist contact form.

JavaScript & Functional Requirements

  • Use fetch() to retrieve medication data from a local JSON file with try...catch for error handling.
  • Render medication cards dynamically (template literals + map()).
  • Filter/search medications by name or category using filter().
  • Form interactions: validate inputs, save medication logs to localStorage, and show confirmation on a result page.
  • Modal dialog for safety tips (accessible: focus trapping + ARIA attributes).
  • ES Modules: split code into fetchData.js, renderCards.js, formHandler.js, modal.js.

Data Sources

  • Local: medications.json with at least 15 entries (fields: name, uses, dosage, sideEffects, warnings, category).
  • Optional External APIs: OpenFDA or DrugBank — use only if allowed and if an API key is available. For the class deliverable, the local JSON file is sufficient.

Scenarios (target audience questions)

  1. "I take multiple medications — how do I check for interactions?"

    This drives content for interaction warnings, a medication log tool, and a clear explanation of when to consult a pharmacist.

  2. "What are the common side effects of this OTC pain reliever and how should I store it at home?"

    This requires clear card content (side effects, storage, and disposal tips) and a modal for quick safety reminders.

  3. "How can I contact a pharmacist or submit a question anonymously?"

    Design the Ask a Pharmacist form to accept questions and optionally provide contact preferences.

Testing & Self-check

Planned manual and automated checks prior to submission:

  • HTML validation: Run W3C Markup Validator on index.html.
  • Accessibility: Use Lighthouse accessibility audit, WAVE, and axe (browser extension). Ensure keyboard navigation and ARIA roles for modal dialogs.
  • Color contrast: Check WCAG contrast for body text and interactive elements (aim for AA for normal text, AAA where possible).
  • SEO & best practices: Add meaningful title, meta description, semantic headings, and descriptive alt attributes for images.
  • Performance: Use Lighthouse; keep JS modular and defer non-critical scripts. Optimize images (WebP where possible) and use srcset for responsive images.
  • Form validation: Client-side validation + clear error messages; test localStorage persistence in multiple browsers.
